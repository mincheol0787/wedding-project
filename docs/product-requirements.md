# 제품 요구사항 문서

## 문제 정의

예비부부는 식전영상, 모바일 청첩장, RSVP 수집, 방명록 관리를 여러 도구로 나누어 처리합니다. 이 과정은 비개발자에게 복잡하고, 디자인 품질을 일정하게 유지하기 어렵고, 수정 요청이 반복될수록 시간이 많이 듭니다.

## 제품 목표

- 하나의 웨딩 프로젝트에서 식전영상과 모바일 청첩장을 함께 제작합니다.
- 비개발자가 템플릿을 선택하고 정보를 입력하는 방식으로 완성물을 만들 수 있게 합니다.
- MVP 이후 프리미엄 템플릿, 고화질 렌더링, 커스텀 도메인, 광고 제거 같은 유료화 옵션을 붙일 수 있게 설계합니다.

## 주요 사용자

- 예비부부: 직접 청첩장과 영상을 만들고 공유합니다.
- 웨딩 플래너: 여러 고객의 프로젝트를 관리합니다.
- 관리자: 템플릿, 렌더링 실패, 신고/숨김 콘텐츠를 관리합니다.

## 핵심 사용자 시나리오

1. 사용자는 회원가입 후 새 웨딩 프로젝트를 생성합니다.
2. 프로젝트 안에서 식전영상과 모바일 청첩장을 각각 편집합니다.
3. 모바일 청첩장은 예식 정보, 사진, 지도, RSVP, 방명록을 포함한 공개 링크로 배포합니다.
4. 식전영상은 사진, 문구/가사, 음악을 넣고 템플릿 기반 mp4로 렌더링합니다.
5. 사용자는 렌더링 결과와 청첩장 링크를 공유합니다.

## MVP 기능

- 계정 생성과 로그인
- 웨딩 프로젝트 생성
- 모바일 청첩장 공개 페이지
- RSVP와 방명록 작성
- 미디어 업로드 메타데이터 저장
- 영상 장면과 가사 데이터 저장
- 렌더링 job 생성 및 worker 수신
- 관리자 페이지 초안

## 성공 기준

- 사용자가 10분 안에 모바일 청첩장 초안을 만들 수 있습니다.
- 공개 청첩장 링크가 로그인 없이 안정적으로 열립니다.
- RSVP와 방명록 데이터가 프로젝트별로 수집됩니다.
- 영상 렌더링 요청이 웹 요청과 분리되어 큐에 안정적으로 등록됩니다.

## 비기능 요구사항

- 서버와 클라이언트 역할을 분리합니다.
- 공개 페이지는 내부 ID를 노출하지 않습니다.
- 개인정보 수집 항목은 명확히 동의받습니다.
- 모든 주요 테이블은 soft delete를 지원합니다.
- 템플릿 설정은 DB와 JSON config로 확장 가능해야 합니다.
