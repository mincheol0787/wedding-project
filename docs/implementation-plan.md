# MVP 범위와 구현 계획

## MVP 범위

MVP는 사용자가 가입 후 하나의 웨딩 프로젝트를 만들고, 모바일 청첩장을 공개 링크로 배포하며, 기본 식전영상을 렌더링 요청할 수 있는 수준을 목표로 합니다.

### 포함

- 이메일 기반 회원가입/로그인
- 웨딩 프로젝트 생성/목록/상세
- 이미지와 음악 업로드를 위한 S3 연동
- 모바일 청첩장 편집 및 `/i/[slug]` 공개 페이지
- RSVP 작성 및 관리자 확인
- 방명록 작성 및 숨김 처리
- 식전영상 장면/문구/음악 편집
- BullMQ 기반 렌더링 작업 생성
- Remotion 기본 템플릿 mp4 렌더링
- 관리자 페이지 초안: 사용자, 프로젝트, 렌더링 실패 job 조회

### 제외

- 결제/구독
- 다중 브랜드 템플릿 마켓
- 고급 영상 효과 편집
- 실시간 협업
- 카카오톡 공유 최적화 자동화
- CDN 캐시 무효화 자동화

## 우선순위

### P0: 서비스 뼈대

- Next.js, TypeScript, Tailwind, Prisma 설정
- Auth.js와 Prisma Adapter 연결
- Prisma 마이그레이션
- 공통 레이아웃, 대시보드 라우팅

### P1: 프로젝트와 청첩장

- `WeddingProject` 생성 플로우
- `InvitationProject` 편집 폼
- 공개 청첩장 페이지 `/i/[slug]`
- RSVP, 방명록 작성 API

### P2: 미디어와 영상 편집

- S3 presigned upload
- `MediaAsset` 등록
- 영상 장면 타임라인 편집
- 가사/문구 세그먼트 저장

### P3: 렌더링

- `RenderJob` 생성
- BullMQ Worker 실행
- Remotion composition에 DB 입력 연결
- mp4 결과 S3 업로드와 상태 갱신

### P4: 관리자와 유료화 준비

- 관리자 페이지 초안
- 템플릿 관리
- 실패 렌더링 재시도
- 프리미엄 템플릿 구분
- 결제 모델을 붙일 수 있는 플랜/권한 확장

## 권장 구현 순서

1. Prisma schema 검증 및 초기 마이그레이션
2. Auth.js 설정과 로그인 화면
3. 대시보드 프로젝트 생성
4. 모바일 청첩장 편집/공개 페이지
5. RSVP/방명록 저장
6. S3 업로드
7. 영상 편집 데이터 모델 연결
8. BullMQ 렌더링 job 생성
9. Remotion worker 구현
10. 관리자 초안
