# Changelog

## 2026-04-29
- 타 모바일 청첩장 서비스의 RSVP/하객 관리 흐름을 참고해 청첩장 수정 화면에 하객 응답 관리 센터를 추가했습니다.
- 참석 응답 요약, 신랑측/신부측 인원 분리, 최근 응답 테이블, CSV 다운로드, 방명록 공개/숨김 관리 기능을 추가했습니다.
- 모바일 청첩장 기능 점검 결과와 다음 우선순위를 `docs/mobile-invitation-audit.md`에 정리했습니다.
- 영상 제작 요청이 웹 요청을 붙잡지 않도록 `RenderJob` 생성과 BullMQ enqueue를 분리하고, Redis 연결이 없거나 느린 경우에도 서비스가 멈추지 않도록 900ms 비차단 큐 등록 구조로 변경했습니다.
- Remotion worker 시작 시 DB에 남아 있는 `QUEUED` 작업을 다시 BullMQ에 등록하도록 보강해, Redis 장애나 worker 재시작 후에도 대기 작업을 이어서 처리할 수 있게 했습니다.
- Vercel 배포에 로컬 `.env`가 포함되지 않도록 `.vercelignore`를 추가하고, 영상 제작 worker 운영 문서를 정리했습니다.
- 식전영상 템플릿을 PPT형 슬라이드에서 시네마틱 웨딩 필름형 구성으로 재설계하고, 장면별 카피/사진 레이아웃/필름 질감/오프닝/클로징을 추가했습니다.
- 업로드 사진과 음악이 서버 렌더링에서 `blob:` 임시 주소로 깨지지 않도록 렌더 입력에 데이터 URL을 함께 저장하도록 개선했습니다.
- 서버에서 사진 원본을 찾을 수 없는 기존 작업도 어색한 슬라이드 대신 웨딩 무드의 대체 컷으로 렌더링되도록 fallback 이미지를 개선했습니다.
- 같은 영상 작업을 다시 만들 때 기존 결과 파일의 MediaAsset을 업데이트하도록 바꿔 재제작 충돌을 막았습니다.
- Remotion 영상 렌더러가 Next.js 경로 별칭을 해석하지 못하던 문제를 수정하고, 실제 mp4 생성 검증을 완료했습니다.
- Windows 로컬 worker 실행에서 `npx` 실행과 공백 포함 경로가 실패하던 문제를 보완해 렌더링 명령이 안정적으로 실행되도록 수정했습니다.
- 브라우저 임시 이미지(`blob:` URL)가 서버 렌더링에서 실패하지 않도록 서버 안전 대체 이미지를 적용하고, `mc5320@nate.com` 계정의 최신 영상 제작 요청을 완료 상태로 연결했습니다.

날짜별 주요 수정사항과 배포 관련 변경을 기록합니다.

커밋 메시지 기반으로 갱신하려면 `npm run changelog:update`를 실행하세요.
