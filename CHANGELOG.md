# Changelog

날짜별 주요 수정사항과 배포 관련 변경을 기록합니다.

커밋 메시지 기반으로 갱신하려면 `npm run changelog:update`를 실행하세요.

## 2026-04-23
- 모바일 청첩장 수정 페이지에서 미리보기와 내용 수정 영역을 자르던 `max-height`/내부 스크롤/sticky 조합을 제거해, 브라우저 기본 스크롤로 하단 내용까지 자연스럽게 확인할 수 있도록 수정했습니다.
- 내 작업에서 공개 청첩장 또는 미리보기로 이동할 때 버튼 내부 로딩을 표시하지 않고 즉시 페이지 전환이 시작되도록 개선했습니다.
- 공개 청첩장 갤러리에 썸네일 그리드, 12장 기본 노출, 더 보기/접기, 전체 화면 슬라이드, 좌우 버튼, 키보드 이동, 모바일 swipe를 추가했습니다.
- 청첩장 편집 화면에서 사진 표현 방식을 슬라이드/전체보기/애니메이션 버튼으로 선택하고, 선택 즉시 좌측 미리보기와 공개 페이지 데이터에 반영되도록 정리했습니다.
- 청첩장 사진 목록에 대표 이미지 설정 기능을 추가하고, 대표 이미지를 첫 화면 이미지와 미리보기 커버에 우선 반영하도록 개선했습니다.
- 상단 메뉴에서 `/dashboard/projects/new`의 `new`를 실제 작업 ID처럼 인식하던 문제를 수정해, 청첩장 만들기/영상 만들기 메뉴를 반복 클릭해도 404로 이동하지 않도록 했습니다.
- 현재 페이지와 동일한 상단 메뉴를 다시 누르면 페이지 이동 대신 화면 상단으로 부드럽게 이동하도록 처리했습니다.
- 식전영상 편집 화면을 노래 선택, 사진 업로드, 사진 셋팅, 가사 수정의 4단계 제작 흐름으로 재설계했습니다.
- 식전영상 화면에서 템플릿 선택과 Render JSON 노출을 제거하고, 사용자가 필요한 사진 수와 제작 준비도를 바로 이해할 수 있게 바꿨습니다.
- 영상 제작 상태 문구를 제작 준비 중, 영상 제작 중, 제작 완료, 제작 실패 등 사용자 중심 표현으로 정리하고 진행률 UI를 개선했습니다.
- 영상 제작 완료 알림의 SMS/카카오 확장은 MVP 이후 작업으로 `docs/todo.md`에 기록했습니다.
- 청첩장 편집 페이지를 좌측 미리보기 40%, 우측 내용 수정 60%의 2-column 구조로 재구성해 미리보기가 편집 영역을 덮지 않도록 정리했습니다.
- 청첩장 편집 미리보기와 내용 수정 영역에 각각 독립 스크롤을 적용하고, 모바일에서는 미리보기가 먼저 보이는 흐름으로 안정화했습니다.
- 공개 전 확인과 공유 도구를 별도 좌측 라인에서 제거하고 내용 수정 하단 영역으로 이동했습니다.
- 내용 수정 화면을 클릭하거나 입력 필드에 진입하면 좌측 미리보기가 해당 화면 위치로 부드럽게 이동하도록 scroll sync를 추가했습니다.
- 상단 메뉴를 홈, 내 작업, 청첩장 만들기, 영상 만들기 중심으로 정리하고 신규 기능에는 NEW 배지를 표시했습니다.
- 청첩장 편집 로딩 화면을 2-column skeleton으로 바꿔 데이터 로딩 중에도 화면 구조가 즉시 보이도록 개선했습니다.
- 모바일 청첩장 편집 화면을 좌측 컨트롤, 중앙 편집, 우측 미리보기 3-column 구조로 고정하고 각 영역에 독립 스크롤을 적용해 미리보기가 본문을 덮지 않도록 개선했습니다.
- 청첩장 라이브 미리보기의 검은 외곽 배경을 제거하고 ivory/warm gray 톤의 독립 패널과 부드러운 그림자로 정리했습니다.
- 식전영상에 샘플 음악 분위기 프리셋(K-pop 2종, Pop 1종)과 저작권 안전한 데모 자막 구조를 추가했습니다.
- Remotion 입력 JSON에 `musicPreset`, `translation`, 자막 스타일, `cross-dissolve` 전환 타입을 추가하고 영어/번역 2줄 자막 렌더링을 지원했습니다.
- 모바일 청첩장 편집 화면을 좌측 옵션 레일, 중앙 편집 폼, 우측 sticky 미리보기 패널의 3-column 구조로 확장했습니다.
- 청첩장 편집에 자동저장 시간 표시, undo/redo, 공개 전 체크리스트, 공개 URL 복사, 카카오 공유, QR 코드 표시 UI를 추가했습니다.
- 캘린더 헤더를 내부 정렬형 구조로 재정비하고 오늘 일정, 다음 일정 D-day, 카테고리 색상 표시를 추가했습니다.
- 식전영상 편집에 약 1분 길이의 봄 감성 샘플 구성 버튼을 추가하고, 렌더링 요청 전 사진/문구/영상 길이 검증 체크리스트를 적용했습니다.
- 렌더링 작업 목록에서 대기/진행 작업 취소, 실패/취소 작업 재렌더링, 작업 삭제를 할 수 있도록 서버 액션과 UI를 연결했습니다.
- 취소되거나 삭제된 렌더링 작업은 worker가 뒤늦게 실행되어도 완료 처리하지 않도록 실행기 검증을 추가했습니다.
- Redis 큐 연결을 렌더링 요청 시점으로 지연해, Redis 미실행 상태에서 빌드 시 불필요한 연결 오류 로그가 발생하지 않도록 정리했습니다.
- 로그인 상태 플로팅 CTA 문구를 "진행 중인 작업"과 "최근 작업 이어가기" 중심으로 자연스럽게 수정했습니다.
- 사용자에게 노출되는 주요 화면의 "프로젝트" 표현을 "내 작업", "작업", "청첩장 만들기" 중심으로 정리했습니다.
- `/i/sample` 레이아웃을 왼쪽 모바일 미리보기와 오른쪽 섹션 카드 구조로 안정화하고, 불필요한 "보기" 버튼을 제거했습니다.
- 청첩장 지도 영역의 카카오 지도 SDK 로딩, 마커 표시, 로딩/실패 상태 UI를 개선했습니다.
- 청첩장 갤러리를 여백 중심의 카드 그리드와 라이트박스 구조로 재설계했습니다.

- 연락처 섹션을 추가하고 신랑/신부 연락처 저장, 미리보기, 공개 페이지 표시를 연결했습니다.
